# Volume 与 PV/PVC（存储）

容器的总是临时的，当容器被重启或删除时，容器的数据便会消失。和Docker类似，k8s也提供了Volume（数据卷）和PVC（Persistent Volume Claim，持久卷声明）来实现数据的持久化。

## Volume（卷）

Volume本质上是一个目录，存在于宿主机上，并且被挂载到容器中。

### Volume常用类型

- 临时性卷

  - emptyDir

    emptyDir可以翻译成空目录，当 Pod 被分配到 Node 上时创建，初始状态为空。Pod中的所有目录都可以读写emptyDir卷。如果容器崩溃重启，emptyDir 中的数据会保留；但如果 Pod 被移出 Node，数据将被永久删除。

  - hostPath

    hostPath可以把宿主机 Node 操作系统上的文件或目录挂载到 Pod 中。
