# Pod的使用与管理

pod是k8s中最小的调度单位，可以理解为容器组。通常一个pod只装载一个容器，以方便解耦。除非需要多个容器在一起的服务。

---

## 创建Pod

创建Pod的方式有2种，使用`kubectl run` 命令可以创建Pod。另一种方式需要编写一个YAML文件，在YAML中会另讲。

```bash
[root@k8s-master qyc]# kubectl run nginx-demo --image=nginx
pod/nginx-demo created
```

---

## 查看Pod

使用`kubcetl get pod` 命令可以查看pod状态

```bash
pod/nginx-demo created
[root@k8s-master qyc]# kubectl get pods nginx-demo
NAME         READY   STATUS    RESTARTS   AGE
nginx-demo   1/1     Running   0          44m
```

使用`kubectl get  pods` 命令可以查看所有pod的状态。

```bash
[root@k8s-master qyc]# kubectl get  pods
NAME         READY   STATUS              RESTARTS   AGE
nginx-demo   0/1     ContainerCreating   0          176m
```

### pod的状态

- 常见正常状态：

`Pending`：Pod 已被 Kubernetes 接受,但容器尚未创建

`Running`：Pod 运行中

`Succeeded`：Pod已经成功执行并终止，并且不会再被重启，常见于一次性任务。

`Terminating`：Pod 正在删除过程中

`ContainerCreating` ：容器创建中，长时间为此状态也是不正常的。

- 常见错误状态：

`Unknown`：无法获取 Pod 状态

`ImagePullBackOf`f/`ErrImagePullC`：镜像拉取失败。

`CrashLoopBackOff`：Pod启动后立即崩溃。

`Error`：启动失败。

`Evicted`：Pod 被驱逐。通常为node资源不足。

`CreateContainerConfigError`：无法创建容器配置。

使用`kubectl logs` 命令可以查看Pod日志

```bash
[root@k8s-master qyc]# kubectl logs nginx-demo
/docker-entrypoint.sh: /docker-entrypoint.d/ is not empty, will attempt to perform configuration
/docker-entrypoint.sh: Looking for shell scripts in /docker-entrypoint.d/
……
```

## 使用Pod

使用`kubectl exec [pod名] [命令]` 可以在容器中执行单条命令。

```bash
[root@k8s-master qyc]# kubectl exec nginx-demo pwd
kubectl exec [POD] [COMMAND] is DEPRECATED and will be removed in a future version. Use kubectl exec [POD] -- [COMMAND] instead.
/
```

获取一个交互式的shell可以交互式的执行命令，在`kubectl exec` 中使用一个`-it`参数可以获取一个交互式的shell。

键入`exit`即可退出。

```bash
[root@k8s-master qyc]# kubectl exec -it nginx-demo -- sh
# ls
bin   dev                  docker-entrypoint.sh  home  lib64  mnt  proc  run   srv  tmp  var
boot  docker-entrypoint.d  etc                   lib   media  opt  root  sbin  sys  usr
# pwd
/
# exit
```

## 删除Pod

使用 `kubectl delete pod` 命令可以删除pod。

```bash
[root@k8s-master qyc]# kubectl delete pod nginx-demo
pod "nginx-demo" deleted
```

使用`--force`参数可以强制删除。

```bash
[root@k8s-master qyc]# kubectl delete pod nginx-demo  --force
Warning: Immediate deletion does not wait for confirmation that the running resource has been terminated. The resource may continue to run on the cluster indefinitely.
pod "nginx-demo" force deleted
```
