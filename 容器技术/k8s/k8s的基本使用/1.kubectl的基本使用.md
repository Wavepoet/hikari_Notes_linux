## 集群

使用`kubectl cluster-info` 可以查看控制平面和核心服务的网络地址。

```bash
[root@k8s-master qyc]# kubectl cluster-info
Kubernetes control plane is running at https://cluster-master:6443
CoreDNS is running at https://cluster-master:6443/api/v1/namespaces/kub    e-system/services/kube-dns:dns/proxy

To further debug and diagnose cluster problems, use 'kubectl cluster-in    fo dump'.
```

使用`kubectl get nodes`命令可以列出所有的node。

```bash
[root@k8s-master qyc]# kubectl get nodes
NAME          STATUS   ROLES           AGE    VERSION
k8s-master    Ready    control-plane   2d1h   v1.28.2
k8s-worker1   Ready    <none>          2d1h   v1.28.2
k8s-worker2   Ready    <none>          47h    v1.28.2
---------------------------------------------------------
名称           状态      角色         运行时间    版本
```

## 上下文

在k8s中上下文是一个配置，由集群（Cluster），用户（User）和命名空间（Namespace）三个部分组成。上下文有关的`kubectl config`命令。

使用`kubectl config get-contexts` 命令可以列出所有的上下文配置

```bash
[root@k8s-master qyc]# kubectl config get-contexts
CURRENT   NAME                          CLUSTER      AUTHINFO           NAMESPACE
*         kubernetes-admin@kubernetes   kubernetes   kubernetes-admin

```

使用`kubectl config current-context` 命令可以当前活动的上下文 。

```bash
[root@k8s-master qyc]# kubectl config current-context
kubernetes-admin@kubernetes

```

使用`kubectl config use-context <context-name>` 命令可以切换到指定的上下文。

```bash
[root@k8s-master qyc]# kubectl config use-context  kubernetes-admin@kubernetes
Switched to context "kubernetes-admin@kubernetes".
```

## 资源

使用`kubectl get` 命令可以查看特定的资源。

例如：

```bash
kubectl get services       # 查看 Service
kubectl get deployments    # 查看 Deployment
kubectl get nodes          # 查看集群中的所有节点
kubectl get namespaces     # 查看所有的命名空间 (Namespace)
kubectl get configmaps     # 查看 ConfigMap
kubectl get secrets        # 查看 Secret
kubectl get ingress        # 查看 Ingress
```

`-n`：可以选择命名空间查看

`-a`：查看全部命名空间的资源

`-o`：列出更多信息

使用`kubectl get all -A` 命令可以查看所有资源的高级预览

```bash

calico-apiserver       pod/calico-apiserver-84757c7f9f-m6vhg            1/1     Running   0             119m
……
AMESPACE              NAME                                      TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)                  AGE
calico-apiserver       service/calico-api                        ClusterIP   10.96.9.169     <none>        443/TCP                  119m
calico-system          service/calico-kube-controllers-metrics   ClusterIP

```

使用 `kubectl describe [资源类型]  [ 资源名称]`可以查看特定资源的详细、人类可读的信息，算是较为全能的命令了。

```bash
[root@k8s-master qyc]# kubectl describe nodes  k8s-worker2
Name:               k8s-worker2
Roles:              <none>
Labels:             beta.kubernetes.io/arch=amd64

```
