# k8s Namespace

在docker中有讲过， 命名空间是由linux内核所提供的资源隔离机制。在k8s中也有一个命名空间，但是命名空间与Linux命名空间并不是一个东西。K8s 的命名空间是对集群内资源的逻辑分区，并不依赖 Linux 内核机制。在这就讲讲k8s的命名空间及相应命令。

### k8s 命名空间相关命令

ns是Namespace（命令空间）的缩写，使用`kubectl get ns` 命令可以列出所有的Namespace。

```bash
[root@k8s-master qyc]# kubectl get ns
NAME                   STATUS   AGE
calico-apiserver       Active   117m
calico-system          Active   117m
default                Active   2d2h
kube-node-lease        Active   2d2h
kube-public            Active   2d2h
kube-system            Active   2d2h
kubernetes-dashboard   Active   3h2m
tigera-operator        Active   47h
--------------------------------------
名称                    状态    运行时间
```

使用`kubectl create ns [ns名称]`可以创建ns。

```bash
[root@k8s-master qyc]# kubectl create ns namespacedemo
namespace/namespacedemo created
[root@k8s-master qyc]# kubectl get ns
NAME                   STATUS   AGE
……
namespacedemo          Active   9s
```

### 创建命名空间

查看命名空间

```bash
[root@k8s-master qyc]# kubectl config get-contexts
CURRENT   NAME                          CLUSTER      AUTHINFO           NAMESPACE
*         kubernetes-admin@kubernetes   kubernetes   kubernetes-admin
```

为空表示当前默认命名空间位default
