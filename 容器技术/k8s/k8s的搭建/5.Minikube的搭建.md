# Minikube的搭建

k8s的安装还是挺繁琐的，Minikube提供一个轻量化和便携的k8s实现，它的核心目标是让你能够在本地计算机上轻松运行和体验k8s,拿来做一些测试，学习……还是挺不错的。

推荐使用Docker或KVM2作为驱动运行Minikube。

官网：https://minikube.sigs.k8s.io/docs/

中文：https://minikube.kubernetes.ac.cn/docs/

## Linux安装

安装

```bash
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-latest.x86_64.rpm
sudo rpm -Uvh minikube-latest.x86_64.rpm
```

启动集群

```bash
minikube start --driver=docker
```

## windos上安装

使用Windows自带的包管理工具``winget``可以进行一键安装(需要docker)

```PowerShel
winget install minikube
```

启动集群

```PowerShel
minikube start --driver=docker
```

## Minikube的基本使用

### 生命周期管理

- 查看集群状态

```bash
minikube status
```

- 暂停集群（不释放资源）

```bash
minikube pause
```

- 停止集群

```bash
minikube stop
```

- 彻底销毁集群

```bash
minikube delete
```

- 打开k8s的dashboard（web）（安装前需要下载）

```bash
minikube dashboard
```

截至2026年1月， Kubernetes Dashboard 项目正式归档并停止维护。

可以使用Headlamp作为dashboard的替代

- 开启Headlamp服务

```bash
minikube addons enable headlamp
minikube addons enable metrics-server
```

- 启动并访问web

```bash
minikube service headlamp -n headlamp
```

- 获取令牌,时长为24小时。

```bash
kubectl create token headlamp --duration 24h -n headlamp
```

### 与集群的交互

如果系统上有``kubectl``，可以直接使用。

```bash
qyc@qycLinux:~$ kubectl get nodes 
NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   35m   v1.35.1
```

minikube中内置了个kubectl。

可以使用``minikube kubectl``命令来进行交互。但默认没有补全，还是推荐下个kubectl。

```bash
qyc@qycLinux:~$ minikube kubectl get nodes
NAME       STATUS   ROLES           AGE   VERSION
minikube   Ready    control-plane   38m   v1.35.1
```

一键浏览器中打开web服务

```bash
minikube service <pod>
```

仅获取访问 URL,不会打开浏览器，可以用于API测试。

```bash
minikube service <pod> --url
```
