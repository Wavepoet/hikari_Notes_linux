# k8s防火墙策略配置

在前面的安装中，为了简化流程，我直接将防火墙与SELinux直接关闭了，在实际生产当中这是很危险的。在这里额外说明一下k8s集群所需要的端口以及相应的配置。算是对k8s集群部署的补充吧。

---

开启防火墙。

```bash
systemctl enable firewalld
systemctl start firewalld
```

将集群节点IP配置到可信区域内。

```bash
firewall-cmd --permanent --zone=trusted --add-source=192.168.24.174
firewall-cmd --permanent --zone=trusted --add-source=192.168.24.175
firewall-cmd --permanent --zone=trusted --add-source=192.168.24.176
```

## master配置

```bash
#开放端口
firewall-cmd --zone=public --add-icmp-block-inversion --permanent
firewall-cmd --zone=public --add-port=6443/tcp --permanent
firewall-cmd --zone=public --add-port=2379-2380/tcp --permanent
firewall-cmd --zone=public --add-port=10250/tcp --permanent
firewall-cmd --zone=public --add-port=10251/tcp --permanent
firewall-cmd --zone=public --add-port=10252/tcp --permanent
firewall-cmd --zone=public --add-port=30000-32767/tcp --permanent
#firewalld
firewall-cmd --add-masquerade --permanent
firewall-cmd --reload
firewall-cmd --list-all --zone=public
```

端口功能：

| 端口 | 使用端口组件 | 用途 |
| --- | --- | --- |
| TCP:6443 | kube-apiserver | API接口 |
| TCP:2379 | etcd | 响应客户端请求 |
| TCP:2380 | etcd | etcd集群内部成员互相通信 |
| TCP:10250 | kubelet | API Server通过这个端口连接到kubelet |
| TCP:10251 | kube-scheduler | scheduler进行康检查和性能指标服务，通常由监控系统访问。 |
| TCP:10252 | kube-controller-manager | 主要用于健康检查和性能指标的监控 |

## worker配置

```bash
#开放端口
firewall-cmd --zone=public --add-icmp-block-inversion --permanent
firewall-cmd --zone=public --add-port=10250/tcp --permanent
firewall-cmd --zone=public --add-port=30000-32767/tcp --permanent
#firewalld
firewall-cmd --add-masquerade --permanent
firewall-cmd --reload
firewall-cmd --list-all --zone=public
```

端口功能：

| TCP:10250 | kubelet | API Server通过这个端口连接到kubelet |
| --- | --- | --- |
| UDP:30000-32767 | --- | NodePort 类型的 Service 暴露端口 |

### 网络插件（所有节点）

根据使用的网络插件来判断使用的是什么端口。我使用的是Calico。

```bash
# Flannel（VXLAN）
firewall-cmd --permanent --add-port=8472/udp
# Calico（BGP）
firewall-cmd --permanent --add-port=179/tcp

```

### 重新加载配置

```bash
iptables -D INPUT -j REJECT --reject-with icmp-host-prohibited
firewall-cmd --reload
```
