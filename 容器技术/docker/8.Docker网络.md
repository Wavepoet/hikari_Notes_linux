# Docker网络

Docker 网络负责容器与其他容器和外界网络的通信。

容器默认启用网络功能，一个容器可以连接到多个网络。容器对它连接的网络类型没有信息，也无法判断其同伴是否也是 Docker 工作负载。容器只看到一个带有 IP 地址、网关、路由表、DNS 服务和其他网络细节的网络接口。

Docker为其提供了五种网络驱动程序分别是：

bridge（桥接模式）（默认使用）

host（主机模式）

none（无网络）

overlay（跨主机）

ipvlan

macvlan

## Dokcer网络的基本知识

docker默认使用桥接模式，查看网络信息可以看见有一张名为docker0的虚拟网卡，这张网卡便是docker默认桥接所用的网卡。

```bash
[root@localhost qyc]#  ip addr
……

    link/ether 2e:a6:72:96:e4:36 brd ff:ff:ff:ff:ff:ff

478ebf12c5cb   host      host      local
c5df6e89db39   none      null      local
```

## Docker网络的使用

### 端口映射

在使用`docker run`命令启动容器时，使用`-p`和`-P`参数可以指定端口映射。

```bash
[root@localhost qyc]# docker run -d --name nginx1 -p 8080:80 nginx:latest
5b8ce4b7876421c08fc6dfb5e2288225a3d8e7daa9ea3f7ade4416835c8a5374
[root@localhost qyc]# docker run -d --name nginx2 -P  nginx:latest
99a56a5228062a6571f6eee6d762c5febb99a02b8b444cca58dbb1e189c2445c
[root@localhost qyc]# docker ps
CONTAINER ID   IMAGE          ……   PORTS                                       NAMES
99a56a522806   nginx:latest   ……   0.0.0.0:32768->80/tcp, [::]:32768->80/tcp   nginx2
5b8ce4b78764   nginx:latest   ……   0.0.0.0:8080->80/tcp, [::]:8080->80/tcp     nginx1

```

两种方式的区别是:

- **P：**是容器内部端口**随机**映射到主机的端口。
- **p：**是容器内部端口绑定到**指定**的主机端口。

也可以使用-p参数指定端口映射的ip地址以及指定映射使用的传输层协议。

```bash
[root@localhost qyc]# docker run -d  \
--name nginx1 \
-p 192.168.1.101:8080:80/udp \
-p 192.168.1.101:8080:80/tcp \
nginx:latest
0759e2eb1b48b04abfa3bc04921dbd28e37a3bf85f8c58b2d58e97ae918ac033

[root@localhost qyc]# docker ps
CONTAINER ID   IMAGE           ……   PORTS                                                    NAMES
0759e2eb1b48   nginx:latest    ……   192.168.1.101:8080->80/tcp, 192.168.1.101:8080->80/udp   nginx1
```

`-p 192.168.1.101:8080:80/udp`：将宿主机的192.168.1.101 IP地址的UDP8080端口映射到容器的UDP80端口上。

### 管理网络命令

`docker network ls` ： 查看所有 Docker 网络

`docker network inspect <network_name>`：查看某个网络的详细信息

`docker network create <network_name>`：创建一个自定义网络

`docker network connect <network-name> <container-name>：`将一个正在运行的容器连接到一个网络。

`docker network disconnect <network-name> <container-name>`：将一个容器从网络中断开。

`docker network rm <network_name>`：删除网络（不能有容器连接）

### 指定网络类型

在`docker run`启动容器中使用`-network [容器ID/名称]`可指定使用的网络模式。

在此我指定为host模式，通过筛选详细信息的`NetworkMode`字段可以看到容器的网络模式。

```bash
[root@localhost qyc]# docker run -d \
--name nginx_none \
--network none \
nginx:latest
5ec1d28ed34fe7ed822c75c1e3949cb62b7adde8bb211059135e75b3ce8dad27
[root@localhost qyc]# docker inspect nginx_host | grep NetworkMode
            "NetworkMode": "host",
```

### IPv6地址与IPv4地址

Docker创建网络时，默认分配IPv4的地址，在使用`docker network create`创建网络时可以使用`--IPv6`参数开启IPv6的地址分配，也可以`--ipv4=false`参数禁用IPv4的地址分配。

```bash

[root@localhost qyc]#  docker network create --ipv6 --ipv4=false IPv6
6b760e69d04f5b972426b59ce06027a8416eb4950f90f2024b6c2bb0f57f25d3
[root@localhost qyc]# docker network inspect IPv6 | grep IP
        "Name": "IPv6",
        "EnableIPv4": false,
        "EnableIPv6": true,
        "IPAM": {

```

### DNS

默认情况下容器使用与宿主机相同的DNS服务器，在使用`docker run`创建容器时使用`--dns`参数可以指定DNS服务器。

```bash
[root@localhost qyc]# docker run -d --name nginx nginx:latest
0c1a306126648994e3b8242a282d9d5688ab28d992006d2aca56c28dccb34d6f
[root@localhost qyc]# docker run -d \--name nginx_dns --dns=114.114.114.114  nginx:latest
6994b0fbf2b9d61b048dc9f373fa86d6285015da9ebfd6163834d208d5e76dd1
[root@localhost qyc]# docker exec nginx cat /etc/resolv.conf
……
nameserver 8.8.8.8
nameserver 223.5.5.5
……
[root@localhost qyc]# docker exec nginx_dns  cat /etc/resolv.conf
……
nameserver 114.114.114.114
……

```

在此我创建了2个`nginx`容器，一个未指定DNS，一个则指定了DNS 114.114.114.114。查看容器的/etc/resolv.conf文件可以看到指定的DNS未使用宿主机的DNS，而是使用指定的DNS。

### 多容器的连接（待重写）

一个容器可以连接到多个网络，默认网关由 Docker 选择，并且可能会随着容器网络连接的变化而变化。
注意：Docker无法同时连接 默认网络和自定义网络。

要让 Docker 在创建容器或连接新网络时使用`gw-priority`选择特定的默认网关。默认的 `gw-priority` 为 `0` ，网络中优先级最高的网关将被设置为`1`。

```bash
[root@localhost qyc]# docker run -d \
--network name=network1,gw-priority=1 \
--network IPv6 \
--name nginx_2network \
nginx:latest                                                                                                                     name nginx_2network nginx:latest
55dde287f82553ac02af2d22f2e745630b16e875d58c7c94866e67bdcefa0188

```

在此我事先已创建了network1和IPv6两个网络。
