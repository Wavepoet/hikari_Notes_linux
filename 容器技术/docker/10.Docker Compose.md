# Docker Compose

Docker Compose 主要用于通过一个配置文件来定义和运行多容器的 Docker 应用。例如要运行一个服务，需要Web前端，后端数据库，缓存服务……就可以使用Docker Compose来进行统一的管理。

这个文件默认的标准名称是docker-compose.yml，当然也可以自定义名称，但在执行命令时需要指定文件名称。

docker-compose因版本不同，命令也有所不同。Docker Compose V2版本的命令是`docker compose`，而V1版本的命令是`docker-compose`。

例如：

- Docker Compose V1版本：

`docker-compose up -d <文件名称>` ：一键启动指定文件中所有的服务

`docker-compose down <文件名称>` ：停止并移除指定文件中所有的容器、网络……

- Docker Compose V2版本：

`docker compose up -d <文件名称>` ：一键启动指定文件中所有的服务

`docker compose down <文件名称>` ：停止并移除指定文件中所有的容器、网络……

一般的。Docker Compose使用yaml来编写。在包含Docker Compose.yml目录当中执行docker compose命令即可。

## docker compose命令

在此列举几个常用的docker compose命令：

`docker compose up -d <文件名称>` ：一键启动指定文件中所有的服务

`docker compose down <文件名称>` ：停止并移除指定文件中所有的容器、网络……

`docker compose ps` ：查看当前运行的服务

`docker compose logs <服务名>` ：查看指定服务的日志

``docker-compose restart <服务名>``：重启某个特定的服务。

## docker compose文件示例

现在很多的IDE都有Docker Compose和yaml的插件，提供一键补全模板，当然也可以用AI来生成Docker Compose文件👍。

在这里就只列举一个简单的Docker Compose文件供参考了

这是一个包含``Web应用``和``MySQL数据库``的系统：

```yaml
version: '3.8' # Compose 语法版本

services: # 开始定义各个容器服务
  db: # 数据库服务（服务名可以直接当内部域名用）
    image: mysql:8.0 # 使用的镜像
    restart: always # 总是自动重启
    environment: # 容器内部的环境变量
      MYSQL_ROOT_PASSWORD: root_password # 设定的数据库密码
      MYSQL_DATABASE: wordpress # 启动时自动建库的名字
    volumes: # 数据持久化设置
      - db_data:/var/lib/mysql  # 把数据库文件存在宿主机，防丢失

  wordpress: # Web 网站服务
    depends_on: # 启动顺序
      - db  # 必须等上面的 db 启动后再启动
    image: wordpress:latest # 使用的镜像
    restart: always # 总是自动重启
    ports: # 端口映射 (宿主机:容器)
      - "8000:80"  # 浏览器访问 8000 端口就能看到网站
    environment: # 传给 WordPress 的环境变量
      WORDPRESS_DB_HOST: db:3306 # 数据库地址（直接填上面的服务名 db）
      WORDPRESS_DB_USER: root # 连数据库的账号
      WORDPRESS_DB_PASSWORD: root_password # 连数据库的密码

volumes: # 声明需要用到的数据卷
  db_data: # 对应上面 db 用的持久化数据卷
```
