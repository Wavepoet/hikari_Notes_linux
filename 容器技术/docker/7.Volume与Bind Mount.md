# Volume 与 Bind Mount

容器总是临时性的，如果容器被删除，数据也会消失，Volume (数据卷)和 Bind Mount (绑定挂载)，都是Docker中数据持久化的机制，容器删除时，数据都会得到保留。

不同的是：

Volume是由Docker管理的数据存储机制，由 Docker 引擎在宿主机 `/var/lib/docker/volumes/` 下维护。Volume 不依赖于主机文件系统路径，更易于迁移和备份。

Bind Mount则是直接把宿主机上的目录或文件挂载到容器中。需要宿主机的绝对路径，由宿主机的操作系统进行管理。容器中的文件或目录直接映射到主机上的文件或目录，这意味着对容器内文件或目录的更改会直接反映到主机上，反之亦然。

## Volume的使用

### 创建Volume

使用`docker volume create [Volume卷名称]`命令来创建Volume。

```bash
[root@localhost qyc]# docker volume create Volume1
Volume1
```

### 查看Volume

使用`docker volume ls` 命令查看Volume。

```bash
[root@localhost qyc]# docker volume ls
DRIVER    VOLUME NAME
local     Volume1

```

### 查看Volume的详细信息

使用`docker volume inspect [Volume卷名称]`命令来查看Volume的详细信息

```bash
[root@localhost qyc]# docker volume inspect Volume1
[
    {
        "CreatedAt": "2025-09-01T21:39:01+08:00",
        "Driver": "local",
        "Labels": null,
        "Mountpoint": "/var/lib/docker/volumes/Volume1/_data",
        "Name": "Volume1",
        "Options": null,
        "Scope": "local"
    }
]
```

解析：

`CreatedAt`：创建时间

`Driver`：驱动程序

`Labels`：标签

`Mountpoint`：Volume的在宿主机中的位置

`Name`：Volume的名字

`Options`：选项

`Scope`：范围

### 删除Volume

使用`docker volume rm [Volume卷名称]` 命令对Volume进行删除

注意：如果有容器正在使用Volume，即使使用`-f`参数 也无法删除，需要先停止容器。

```bash
[root@localhost qyc]# docker volume rm Volume1
Volume1
[root@localhost qyc]# docker volume  ls
DRIVER    VOLUME NAME
```

### 使用Volume

在使用`docker run`命令创建容器时使用`-v`参数可以将Volume挂载到容器内，如果此时没有这个Volume，则会自动创建这个Volume。

语法为 `-v [Vloume]:[挂载到容器内路径]`。

```bash
[root@localhost qyc]# docker run -d \
-v Vloume1:/app/logs \
centos:latest \
/bin/sh -c "/bin/echo 'hello world' >> /app/logs/helloworld.txt"

2b724b999acef61407917c0ba298da91b533feccf86fc013a9318ec558a487b9
[root@localhost qyc]# cat  /var/lib/docker/volumes/Vloume1/_data/helloworld.txt
hello world
```

也可以使用`--mount`参数来挂载。`--mount` 的语法更长，但参数更清晰，功能也更强大。
语法为 `--mount type=[挂载类型],source=[数据卷的名称],target=[挂载到容器内路径]`。

```bash
[root@localhost qyc]# docker run -d \
--mount type=volume,source=Vloume1,target=/app/logs \
centos:latest \
/bin/sh -c "/bin/echo 'hikari' >> /app/logs/helloworld.txt"

60b137a8ee30912460c924da33509dec360fa4c345d4338feea0c008546d360b
[root@localhost qyc]# cat  /var/lib/docker/volumes/Vloume1/_data/helloworld.txt
hello world
hikari

```

### 多容器挂载

Volume可以同时挂载在多个容器内，从而实现它们之间的数据共享

```bash
# 1. 创建一个共享的日志 volume
docker volume create app-logs

# 2. 启动应用容器，将日志写入 volume
docker run -d --name app \
--mount type=volume,source=app-logs,target=/app/logs my-app-image

# 3. 启动日志处理容器，从同一个 volume 读取日志
docker run -d --name logger \
--mount type=volume,source=app-logs,target=/logs/to/process my-log-processor-image
```

## Bind Mount 的使用

Bind Mount同Volume一样，也使用`-v` 和 `--mount` 参数来进行挂载。

使用`-v`参数进行挂载，语法为`-v [宿主机的绝对路径]:[容器内的绝对路径]`。

```bash
[root@localhost qyc]# docker run  -d \
-v /home/qyc/qyc/:/app/logs/ \
centos:latest \
/bin/sh -c "/bin/echo 'hello world' >> /app/logs/helloworld.txt"
d3e07b561c5aa8c6a9cbe2cf5113f086f221dcb06dd0189b12d742467b53b133

[root@localhost qyc]# cat /home/qyc/qyc/helloworld.txt
hello world
```

使用`--mount`参数进行挂载，语法为`--mount type=bind,source=[宿主机的绝对路径], target=[容器内的绝对路径]`。

```bash
[root@localhost qyc]# docker run -d  \
--mount type=bind,source=/home/qyc/qyc,target=/app/logs \
centos:latest \
/bin/sh -c "/bin/echo 'hikari' >> /app/logs/helloworld.txt"  
f8760f37c1ad4e5d2919d5afc2a61c30008a9009dcacd1c462de934c5986fe53

[root@localhost qyc]# cat /home/qyc/qyc/helloworld.txt                              
hello world
hikari
```
